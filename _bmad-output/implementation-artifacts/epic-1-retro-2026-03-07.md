## 3. Deep story analysis — patterns

### 3.1 Dev notes and struggles

- **1.1 (Init):** `create-next-app` with `.` failed due to directory name containing capitals ("WealthTracker-1", npm restriction). Workaround: created app in `wealthtracker-temp`, then moved contents to root (rsync), set package name to `wealthtracker-1` in package.json.
- **1.2 (Convex):** `npx convex init` deprecated; Convex CLI directs to `npx convex dev --once --configure=new`. Schema and `convex/` created manually; dependency added via `npm install convex`. README updated with linking instructions.
- **1.3–1.7:** No major implementation struggles recorded. 1.7 had a follow-up from code review: FR12 graceful degradation (noscript static fallback) and formatCurrency refinements.

### 3.2 Recurring themes

- **Tooling drift:** Two stories (1.1, 1.2) hit CLI/tooling changes (create-next-app, Convex init). Documentation and README were updated so future runs are clear.
- **No automated tests:** Architecture defers test runner; all stories used manual verification only. Epics.md suggests optional CI (build/lint), scripted pipeline/sync checks, and E2E for Accumulator — none implemented yet.
- **Architecture adherence:** Single rAF loop (1.7), no Convex in app, shared types (1.3) used by pipeline and sync, failure semantics (no overwrite on validation/read failure) followed.

### 3.3 Breakthroughs and wins

- **Single source of truth for data shape:** Story 1.3 types (`BillionaireEntry`, `WealthTrackerMetadata`, `WealthTrackerData`) were used consistently by pipeline, sync, and app — no shape drift.
- **Accumulator as specified:** One client component, one requestAnimationFrame loop, visibility pause, prefers-reduced-motion, aria-live and labels; formula and data flow matched architecture.
- **Clean data boundary:** Pipeline → Convex → sync → `src/data/` → app; app never touches Convex at runtime.

### 3.4 Technical debt and follow-up

- **Test runner:** Still deferred. Adding Vitest (or Jest) and optional E2E would improve regression safety (called out in epics.md QA table).
- **Pipeline/sync automation:** Optional scripted tests (invalid data → non-zero exit; Convex down → no overwrite) not added.
- **Story file status:** Several stories remain in "review" in their implementation-artifacts files; 1.5 is "complete." Consider standardising "done" and updating sprint-status when that file is introduced.
