## 3. Deep story analysis — patterns

### 3.1 Dev notes and struggles

- **2.1 (Top 10 list):** TopTenList already implemented; verification against AC and small enhancement (per-row `aria-label`). No major struggle.
- **2.2 (YTD total):** YTD computed in Accumulator rAF loop and passed via `onSessionUpdate`; ContextStrip displayed "So far this year." Code review found YTD not visible on first paint (initial state 0); fix: PageClient initializes `ytdTotal` with `computeYtdTotal(entries, DEFAULT_RETURN_RATE)` on mount.
- **2.3 (Relatable comparison):** Single comparison line in HeroSection (first-ranked billionaire + "earns median annual salary every X"). Code review found two comparison lines (Accumulator "typical American" + HeroSection named line); consolidated to one (HeroSection only).
- **2.4 (Methodology & citations):** MethodologySection extended with 3%/5%/7%, sources, "Data as of" via shared `formatDataAsOf`; hero link to `#methodology`. Code review: consistent use of `formatDataAsOf` in Hero, noscript, and Methodology; invalid-date guard in formatter.

### 3.2 Recurring themes

- **Reuse of shared libs:** Every story reused `combinedPassiveIncomePerSecond`, `DEFAULT_RETURN_RATE`, `formatCurrency`/`formatCompact`, and (by 2.4) `formatDataAsOf`. No formula or formatting duplication.
- **Code review as quality gate:** Adversarial code review caught first-paint (YTD), single-comparison consistency, and date-format consistency; all addressed before retro.
- **Semantic HTML and a11y:** Section headings, aria-labels, `role="status"`/`region`, linkable methodology; tabular figures and contrast per UX.

### 3.3 Breakthroughs and wins

- **Single comparison line:** Clear "one shareable takeaway" (FR5) after removing duplicate from Accumulator; HeroSection owns the one named-entity comparison.
- **YTD and session data flow:** Accumulator owns one rAF loop and passes `ytdTotal` with session data; no second timer; initial YTD on mount for first-paint visibility.
- **Shared date formatting:** `src/lib/format-date.ts` with `formatDataAsOf` and invalid-date guard used in HeroSection, MethodologySection, and noscript — consistent "Data as of" everywhere.

### 3.4 Technical debt and follow-up

- **Test coverage:** One E2E (YTD region) added in 2.2; no test runner or broader E2E suite yet. Epic 1 retro action item (test runner / baseline) still open; optional for Epic 3.
- **Story/sprint-status alignment:** All four Epic 2 stories are implementation-complete and reviewed; recommend marking 2-1–2-4 as *done* in sprint-status when ready so Epic 2 can be marked *done* and this retro recorded.
