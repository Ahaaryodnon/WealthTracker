## Project Overview

WealthTracker is a single-page wealth inequality tracker that displays a live Accumulator showing combined passive income of the top 10 billionaires, with relatable comparisons (e.g. median salary multiples). It is a statically exported Next.js site — no server runtime at request time.

## Commands

- `npm run dev` — Start Next.js dev server (localhost:3000)
- `npm run build` — Static export build (output: `out/`)
- `npm run lint` — ESLint (flat config, Next.js core-web-vitals + TypeScript)
- `npm run update-data` — Pipeline: fetch Forbes data via rtb-api, validate, write to Convex (requires `CONVEX_URL`)
- `npm run data:sync` — Sync: read from Convex, write `src/data/billionaires.ts` (requires `CONVEX_URL`)
- `npm run deploy` — Build and deploy `out/` to Cloudflare Pages (requires Wrangler; run `data:sync` first for fresh data)
- `npx convex dev` — Run Convex dev server (watches schema/functions)

Run `data:sync` before production builds to get latest data snapshot. Deploy target: **Cloudflare Pages** (static `out/`). Both pipeline scripts need `CONVEX_URL` from `.env.local`.

## Architecture

### Data Flow

```
Forbes API (rtb-api) → update-data script → Convex DB → data:sync script → src/data/billionaires.ts → Next.js static build
```

The app does **not** use Convex at runtime. It consumes `src/data/billionaires.ts` at build time only. Convex is the canonical store used by pipeline/sync scripts.

### Key Paths

- `src/app/` — Next.js App Router (single page, static export via `output: "export"`)
- `src/data/billionaires.types.ts` — Canonical TypeScript interfaces (`BillionaireEntry`, `WealthTrackerData`)
- `src/data/billionaires.ts` — Static dataset (auto-generated by `data:sync`, do not edit by hand)
- `convex/schema.ts` — Convex table definitions (`billionaires`, `metadata`)
- `convex/data.ts` — Convex mutations/queries (`replaceCanonicalData`, `getCanonicalData`)
- `scripts/update-data.ts` — Pipeline: fetches top 10 from Forbes, validates, writes to Convex
- `scripts/data-sync.ts` — Sync: reads Convex, writes `src/data/billionaires.ts`
- `_bmad-output/planning-artifacts/` — PRD, architecture, UX design, epics
- `_bmad-output/implementation-artifacts/` — Story specs for implementation

### Tech Stack

- **Framework:** Next.js 16 (App Router, static export)
- **Styling:** Tailwind CSS v4 (via `@tailwindcss/postcss`)
- **Database:** Convex (pipeline/sync only, not runtime)
- **Fonts:** Geist Sans + Geist Mono (via `next/font`)
- **Scripts:** tsx for running TypeScript pipeline scripts

### Data Model

Net worth values are stored in **billions**. The rtb-api returns values in millions; the pipeline converts. The `BillionaireEntry` type supports per-source net worth (`forbesNetWorth`, `bloombergNetWorth`) with a computed `netWorth` field. Convex `getCanonicalData` averages when both sources exist, falls back to whichever is available.

Passive income is calculated client-side: `(netWorth * returnRate) / (365 * 24 * 3600)` per second, using return assumptions of 3%/5%/7%.

`@/*` maps to `./src/*` (configured in tsconfig.json).
